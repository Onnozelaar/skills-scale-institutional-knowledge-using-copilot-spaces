# Security Checklist (Lightweight)

Use during planning and pre-release to catch common security gaps.

- [ ] Threat modelling or brief security review completed for new features
- [ ] Dependencies scanned for known vulnerabilities
- [ ] Secrets and credentials not stored in repo
- [ ] Access controls reviewed for new services/endpoints
- [ ] Security tests/scans run in CI and addressed (or tracked)
- [ ] Any high/critical findings triaged with owner and timeline
- [ ] Security Liaison or Security on-call notified for risky changes
