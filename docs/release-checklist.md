# Release Checklist

A concise checklist to follow for every release to reduce risk and ensure observability and rollback plans are in place.

- [ ] All linked PRs merged and pass CI
- [ ] Acceptance criteria met and verified in staging
- [ ] Smoke tests defined and executed in staging
- [ ] Rollback plan documented and tested (if applicable)
- [ ] Release notes drafted and approved
- [ ] Deployment window or communication scheduled
- [ ] Monitoring dashboards and alerts validated
- [ ] Post-release verification steps defined
- [ ] Stakeholders notified of release and key changes
